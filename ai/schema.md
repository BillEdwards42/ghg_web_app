# Knowledge Base Schema

## What This Is
A personal knowledge base Ln{Carbon}, a PWA webapp.

## How It's Organized
- Codes are everywhere except in wiki/, memory/, and ai/
- wiki/ contains the organized wiki, a host of markdown files that contains the interconnections between different codes and files. This will be maintained by you.
- memory/ contains md files that act as consistent memory of the agent. This will be maintained by you also.
- ai/ contains the instructions for the agent. This will be maintained by me.

## Wiki Rules
- Every function gets its own .md file in wiki/. So all the function the app currently has will be a .md file in wiki/, such as login, logout, legal entity selection, location selection, manual entry's category selection, and more.
- Every wiki file starts with a one-paragraph summary.
- Link related codes to the function and each other using [[function-file_name]] format.
- Maintain an INDEX.md in wiki/ that lists every function with a one-line description.
- When you make updates, alter the code, or create new parts, be sure to update the wiki as well.
